MVP：数据库关键词检索

先在后端加一个受控的笔记检索函数，例如 search_notes(query, limit=5)，只查 Note.title / description / content，返回最相关的几篇笔记摘要。

流程是：

用户提问 -> 后端查笔记数据库 -> 把相关笔记片段放进 prompt -> 大模型基于这些内容回答。

这个阶段不让模型直接写 SQL，也不把数据库账号交给模型，只开放一个你写好的“搜索笔记工具”。

进阶：向量检索

给每篇笔记切块，比如每 300-800 字一块，存入 note_embedding 表：

id
note_id
chunk_index
content
embedding
updated_at
用户提问时也转成 embedding，然后找最相似的笔记块。这样可以解决“用户问法和笔记原文不完全一致”的问题，比如用户问“我之前写过关于焦虑的东西吗”，即使笔记里写的是“最近有点内耗”，也能搜出来。

真正变成拾页专属助手

在 RAG 基础上加三类能力：

笔记问答：根据我的笔记回答问题，并引用来源笔记标题。
笔记整理：帮我把最近的笔记归类、总结、生成标签。
个人记忆：保存用户偏好，比如写作风格、常用分类、长期目标，但要单独建 ai_memory 表，不要混在笔记内容里。
